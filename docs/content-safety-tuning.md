# 内容安全「审核严格度」调节说明

> 场景:视频语音转写/图片被阿里云内容安全误判(如引用、玩笑里出现"杀人犯"被判暴力)。本文说明**能调哪些维度、分别在哪里调**。
> 系统里有两套审核并存、取最严:①**阿里云内容安全**(机器内置的黄暴政/违禁策略,标签由阿里云模型给)②你后台配的**「审核规则」**(关键词 + 大模型条件)。下面主要讲第 ① 套怎么调松紧。

---

## 〇、后台「内容安全白名单」页(自助,治误伤最快)★
路径:**管理员后台 → 治理 → 内容安全白名单**。把被误判触发的词(如"杀人犯""杀")加进去 → **命中的词只要全在白名单里,即便阿里云判违规也放行(降级为通过)**,即时生效、不依赖阿里云控制台。
原理:阿里云每次返回命中的具体风险词 `riskWords`;本系统比对——命中的词**全部**落在白名单里 → 判 pass;有一个不在 → 维持原判。端点 `/admin/whitelist`(GET/POST/DELETE),存 `state.json` `content_whitelist`,逻辑在 `content_safety.py` `_apply_whitelist`(纯函数可测)。

## 一、代码侧(本系统,改一处即可,我维护)

### 1. 严格度档位 `AM_CONTENT_SAFETY_MODE`(已上线,当前 = `balanced`)
在服务器 `/opt/assets/.env` 设置(改完需 `docker rm -f assets-api` + `docker run …` 重建容器才生效,`docker restart` 不重读 env):

| 档位 | 文本(语音转写/原文) | 图片/帧 |
|---|---|---|
| `strict` | 严重类(色情/政治/暴力/暴恐/违禁/涉毒/赌博)全部**硬拦 block** | 高危→block、中低→复核 |
| **`balanced`(适中,默认)** | 只硬拦**色情/政治/暴恐**;暴力/违禁/辱骂/广告等 → **转人工复核 review**(不硬拦) | 高危→block、中低→复核 |
| `loose` | 内容安全**从不硬拦**,任何风险一律 → 转人工复核 | 任何风险 → 复核(不硬拦) |

- "杀人犯是句玩笑"这类**暴力词**:`balanced` 下 → 转人工复核(你在审核队列里放行/拦截),不再直接拦。
- 真正的硬拦(色情/政治)在 balanced 下仍然拦。
- 想让内容安全**完全不硬拦、只标记**(硬拦只靠你后台规则)→ 用 `loose`。

实现:`app/infrastructure/content_safety.py` 的纯函数 `_text_verdict / _image_verdict`(标签/风险等级 → 裁定映射,可单测)。要**逐类微调**(比如"违禁也硬拦、但暴力放行"),改 `_BLOCK_SUBSTR` 里对应档位的标签清单即可。

### 2. 其他代码侧旋钮
- **分来源区别对待**:文本比图片宽松(现状:文本 balanced、图片高危仍硬拦)。
- **检测哪些来源 / 关掉**:可只审图片像素、放过转写文本;或整个关掉内容安全(`AM_ENABLE_CONTENT_SAFETY=false`,只留后台规则 + 大模型)。

### ⚠️ 关于"按置信度调阈值"
阿里云**文本审核不返回数值置信度/概率分数**,只给 `labels`(标签)+ `riskLevel`(high/medium/low)+ `riskWords`(命中的具体词)。所以文本没有"置信度阈值"这个旋钮;细调靠档位 + 标签映射 + 阿里云控制台(下面)。**图片**有 risk_level 三档,可按等级调。

## 二、阿里云控制台侧(你在阿里云那边配,最细、最推荐)

**入口**:阿里云控制台 → 产品「内容安全」(域名 `yundun.console.aliyun.com`)。以下路径视你开通的版本可能显示为「机器审核增强版」/「机器审核 V2.0」/「API 违规检测增强版」。

### 1. 自定义词库(对"误伤"最直接,强烈建议先用这个)
路径:**机器审核增强版 → 文本审核 → 规则配置 → 规则管理 → 目标服务 → 操作列「设置词库」**
- **可信名单 / 白名单 / 忽略名单** → 命中即**放行/跳过检测**:把"杀人犯""杀"这类在你语境下无害的词加进去,永远不再被误判。
- **黑名单 / 风险名单** → 命中即**违规拦截**。
- **疑似名单** → 命中转**人工复核**。
- 关键词逻辑:`A&B`(同时出现才命中)、`A~B`(有 A 无 B 才命中,`&` 必须在 `~` 前);每行一个词、单词 ≤50 字、单库 ≤1000 行、单账号 ≤10 万词 / ≤20 个库。
- 生效:约 3 分钟。命中自定义库时 API 返回标签 `C_customized`。

### 2. 定制检测范围(关掉容易误伤的类别)
路径:**规则配置 → 规则管理 → 复制内置服务 → 管理检测规则**(逐项开关要检测的风险类别,如关掉"暴力/不良/宗教")→ 得到一个自定义 serviceCode → 告诉我,我在 `content_safety.py` 把 `service="comment_detection"` 换成它即可。

### 3. 换检测场景 serviceCode
文本可选场景:`comment_detection`(公聊评论,现在用的)、`chat_detection`(私聊)、`nickname_detection`(昵称)、`ai_art_detection`(AIGC)、`ad_compliance_detection`(广告法)、`pgc_detection`(教育)。不同场景灵敏度不同。

### 4. 大模型增强(减少语境误伤)
路径:**文本审核 → 规则配置** 里对应服务的「大模型增强」开关。开启后在小模型结果上再叠一层大模型判定(更懂"这是引用/玩笑"),3–5 分钟生效。

### 5. 查看支持的标签 / 结果查询
- 查看标签:规则管理页「查看标签」。
- 结果查询:文本审核 → 结果查询(看命中的标签、riskWords、时间)。

---

## 三、建议(针对误伤)
**适中档(已上线)+ 阿里云控制台把常被误伤的词加「可信名单」+ 开「大模型增强」**,三管齐下基本不会再把引用/玩笑判违规。要更松,把档位切 `loose`(只标记、全转人工)。

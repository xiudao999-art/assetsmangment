# ADR-0005:内容安全审核策略(零漏放)

- 状态:已接受 · 日期:2026-07-09 · 关联:REQ-501/502/503

## 决策
每条物料生成后必过阿里云内容安全(画面+文本),写 `audit_result`。**默认态为 review(不可放行)**;仅 pass 可检索/下载;block 永久不可检索;超时/失败 → review 进人工复核。OSS 违规扫描兜底存量。

## 理由
"零漏放"是硬质量目标;默认不放行 = fail-safe,避免审核失败被当作通过。

## 后果
- 搜索/下载查询必带 `audit_status='pass'` 过滤(见 [[0007-hnsw-index]] 的查询)。
- 内容安全回归集作为闭环④的常驻传感器。
